Slideshow

Note: The people behind Satori are not fans of carousels, but recognize that it's a popular feature in web design. Please be aware that hiding content presents major usability issues, you should consider alternative designs before using this feature as a carousel.

Satori's slideshow widget handles the traditional gallery slideshow that is common throughout the web, in addition to carousels.

The left and right arrow keys are bound to trigger the scroll events.

Slideshow Properties (data-*)

Timer
The timer attribute determines the time between automatic scrolling in milliseconds. Default is 0 (no automatic scrolling).
Visible-Slides
The number of slides that are visible at a given time. Default is 1.
Step
The number of slides that move during each move event; this is most likely only needed when there are more than one visible slides. Default is 1.
Loop
Allows the slideshow to repeat previous elements. Default is false.
Bind-Key-Events
Binds events to left arrow, right arrow, and escape keys. Default is true.
Show Arrows
Set visibility left and right arrows. Default is true.
Show Controls
Set visibility of controls (left and right arrows in a box, along with the number of slides). Default is false.
Highlight Active
Adds emphasis to the center most visible slide; if the number of visible slides is even, it will take the two most center visible slides and error to the left. In laymen's terms, if there are two visible slides, the active one will be the first slide. Default is false.

Markup Example

<div class="container" data-role="slideshow">
	<div data-role="slide" class="bg-primary">
		<p>Slide ###</p>
	</div>
	<div data-role="slide" class="bg-alternate">
		<p>Slide ###</p>
	</div>
	<div data-role="slide" class="bg-secondary">
		<p>Slide ###</p>
	</div>
</div>
